01. Pengantar Extension Methods

Extension Methods merupakan fitur pada bahasa Dart yang digunakan untuk menambahkan method atau fungsi baru pada tipe data yang sudah ada seperti String, List, dan lainnya.

Dengan fitur ini, programmer tidak perlu mengubah kode asli dari tipe data tersebut sehingga kode tetap aman dan lebih rapi.

Extension Methods mulai tersedia sejak Dart versi 2.7 dan membantu penulisan kode menjadi lebih mudah dipahami.

Analogi sederhananya seperti memberikan kemampuan baru pada sebuah benda tanpa harus mengubah bentuk asli benda tersebut.

DART SOURCE CODE
// Nama  : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

extension NameFormatter on String {

  String makeTitleCase() {

    if (isEmpty) {
      return this;
    }

    return this[0].toUpperCase() +
        substring(1).toLowerCase();
  }
}

void main() {

  var username = 'uliya';

  print(username.makeTitleCase());
}

02.Masalah Tanpa Extension Methods

Sebelum adanya Extension Methods, programmer biasanya menggunakan utility class untuk menambahkan fungsi pada tipe data tertentu.

Cara tersebut membuat penulisan kode menjadi lebih panjang dan kurang praktis karena method harus dipanggil melalui class tambahan.

Selain itu, penggunaan utility class juga membuat kode terlihat kurang natural dan sedikit lebih sulit dibaca.

DART SOURCE CODE
// Nama  : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

class TextHelper {

  static String makeUpper(String value) {

    if (value.isEmpty) {
      return value;
    }

    return value[0].toUpperCase() +
        value.substring(1);
  }
}

void main() {

  var studentName = 'uliya';

  var result =
      TextHelper.makeUpper(studentName);

  print(result);
}

03.Solusi dengan Extension Methods

Extension Methods digunakan sebagai solusi agar penambahan fungsi pada tipe data dapat dilakukan dengan lebih mudah dan rapi.

Dengan fitur ini, method dapat langsung dipanggil dari object tanpa perlu menggunakan utility class tambahan.

Penggunaan Extension Methods membuat kode terlihat lebih natural, singkat, dan mudah dipahami oleh programmer.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

extension TextExtensions on String {

  String firstUpper() {

    if (isEmpty) {
      return this;
    }

    return this[0].toUpperCase() +
        substring(1);
  }

  bool get hasEmailSymbol =>
      contains('@');

  String reverseText() {

    return split('').reversed.join();
  }
}

void main() {

  print('uliya'.firstUpper());

  print('uliya@gmail.com'
      .hasEmailSymbol);

  print('dart'
      .reverseText());
}

04.Sintaks Extension Methods

Penulisan Extension Methods pada Dart menggunakan keyword extension yang diikuti nama extension dan tipe data target yang akan ditambahkan method baru.

Sintaks dasar Extension Methods yaitu: extension NamaExtension on TipeData.

Di dalam extension dapat ditambahkan method, getter, setter, maupun operator sesuai kebutuhan.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension CalendarExtensions on DateTime {

  String fullDate() {

    return '$day/$month/$year';
  }

  bool get isCurrentDay {

    var current = DateTime.now();

    return year == current.year &&
        month == current.month &&
        day == current.day;
  }
}

void main() {

  var today = DateTime.now();

  print(today.fullDate());

  print(today.isCurrentDay);
}

05.Tipe yang Bisa Di-extend

Extension Methods dapat digunakan pada berbagai tipe data yang ada di Dart, baik tipe bawaan maupun tipe buatan sendiri.

Beberapa tipe data yang dapat menggunakan extension yaitu built-in types seperti String, int, double, List, dan Map.

Selain itu, extension juga dapat diterapkan pada custom class, generic types seperti List<T> dan Map<K,V>, serta nullable types seperti String? dan int?.

Dengan adanya fitur ini, programmer dapat menambahkan fungsi baru pada berbagai jenis tipe data sesuai kebutuhan.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension ScoreExtensions on List {

  int get totalScore =>
      fold(0, (a, b) => a + b);

  double get averageScore {

    if (isEmpty) {
      return 0;
    }

    return totalScore / length;
  }
}

void main() {

  var scores = [80, 85, 90, 95];

  print(scores.totalScore);

  print(scores.averageScore);
}

06.Extension untuk Null Safety

Extension Methods juga dapat digunakan pada tipe data nullable untuk membantu penanganan nilai null dengan lebih aman.

Fitur ini biasanya digunakan pada tipe seperti String? atau int? agar program tidak mudah mengalami error saat data bernilai null.

Dengan menggunakan extension untuk null safety, programmer dapat membuat kode menjadi lebih aman, rapi, dan mudah dibaca.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

extension UserExtensions on String? {

  String safeText() {

    return this ?? 'Tidak ada data';
  }

  bool get isDataEmpty {

    return this == null ||
        this!.isEmpty;
  }

  String defaultName(String name) {

    return this ?? name;
  }
}

void main() {

  String? username;

  print(username.safeText());

  print(username.isDataEmpty);

  print(username.defaultName('Guest'));
}

07.Praktik 1 – Utilities untuk Dart Types

Pada praktik ini, Extension Methods digunakan untuk menambahkan utility atau fungsi tambahan pada tipe data bawaan Dart.

Beberapa contoh fungsi yang dapat dibuat yaitu format angka, pengecekan range nilai, dan operasi perhitungan sederhana.

Dengan adanya utilities tersebut, penggunaan tipe data menjadi lebih praktis dan membantu mempercepat penulisan kode program.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

extension MoneyExtensions on num {

  String toRupiahFormat() {

    return 'Rp${toStringAsFixed(0)}';
  }

  bool betweenNumber(num min, num max) {

    return this >= min &&
        this <= max;
  }

  num squareNumber() {

    return this * this;
  }
}

void main() {

  var totalMoney = 250000;

  print(totalMoney.toRupiahFormat());

  print(7.betweenNumber(1, 10));

  print(6.squareNumber());
}

08.Generic Extensions

Generic Extensions merupakan extension yang dapat digunakan untuk berbagai tipe data dengan menggunakan generic type.

Fitur ini membuat extension menjadi lebih fleksibel karena tidak hanya digunakan pada satu tipe data tertentu saja.

Contoh generic type yang sering digunakan yaitu List<T> dan Map<K,V> sehingga extension dapat dipakai kembali pada berbagai jenis data.

Dengan Generic Extensions, penulisan kode menjadi lebih efisien dan mudah digunakan kembali.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension CollectionExtensions
    on List {

  T? getSafe(int index) {

    if (index < 0 ||
        index >= length) {

      return null;
    }

    return this[index];
  }

  List removeDuplicate() {

    var finalData = [];

    for (var item in this) {

      if (!finalData.contains(item)) {
        finalData.add(item);
      }
    }

    return finalData;
  }
}

void main() {

  var data =
      [1, 1, 2, 3, 3, 4];

  print(data.removeDuplicate());
}

09.Unnamed Extensions

Unnamed Extensions adalah extension tanpa nama yang digunakan hanya untuk kebutuhan di dalam file yang sama.

Jenis extension ini biasanya dipakai untuk helper sederhana sehingga tidak perlu diakses dari file lain.

Penggunaan unnamed extension juga membantu mengurangi kemungkinan terjadinya konflik nama extension.

Dengan cara ini, kode dapat menjadi lebih ringkas dan rapi untuk penggunaan internal program.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension on DateTime {

  String get simpleDate {

    return '$day-$month-$year';
  }
}

void main() {

  var now = DateTime.now();

  print(now.simpleDate);
}

10.Konflik Nama Extension

Konflik nama extension dapat terjadi ketika terdapat beberapa extension yang memiliki nama method yang sama pada tipe data yang sama.

Kondisi ini dapat menyebabkan program menjadi ambigu karena Dart tidak dapat menentukan method mana yang harus digunakan.

Untuk mengatasi masalah tersebut, programmer dapat menggunakan import prefix atau show dan hide saat melakukan import file.

Dengan cara tersebut, penggunaan extension menjadi lebih jelas dan terhindar dari konflik nama method.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension StudentText on String {

  String greeting() {

    return 'Halo Mahasiswa';
  }
}

extension UserText on String {

  String greeting() {

    return 'Halo User';
  }
}

void main() {

  var text = 'Uliya';

  print(text);
}

11.Extension untuk Custom Classes

Extension Methods tidak hanya dapat digunakan pada built-in types, tetapi juga bisa diterapkan pada custom classes atau class buatan sendiri.

Dengan fitur ini, programmer dapat menambahkan fungsi baru pada class tanpa harus mengubah kode asli dari class tersebut.

Penggunaan extension pada custom classes membantu membuat kode menjadi lebih terstruktur dan mudah dikembangkan.

Selain itu, fitur ini juga mempermudah penambahan utility atau fitur tambahan sesuai kebutuhan program.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

class Item {

  final String itemName;

  final double itemPrice;

  Item(this.itemName, this.itemPrice);
}

extension ItemExtensions on Item {

  Item discountPrice(double percent) {

    var newPrice =
        itemPrice -
        (itemPrice * percent / 100);

    return Item(itemName, newPrice);
  }

  bool get expensiveItem {

    return itemPrice > 500000;
  }
}

void main() {

  var bag =
      Item('Tas Kuliah', 700000);

  var cheapBag =
      bag.discountPrice(20);

  print(cheapBag.itemPrice);

  print(bag.expensiveItem);
}

12.Validator Extensions

Validator Extensions digunakan untuk membantu proses validasi data dengan lebih mudah dan praktis menggunakan Extension Methods.

Validasi dapat diterapkan pada berbagai tipe data seperti String maupun int sesuai kebutuhan program.

Beberapa contoh validasi yang sering digunakan yaitu validasi email, password, nomor telepon, serta pengecekan angka positif atau ganjil dan genap.

Dengan Validator Extensions, penulisan kode validasi menjadi lebih singkat, rapi, dan mudah dipahami.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

extension ValidationExtensions
    on String {

  bool get validEmail {

    return RegExp(
      r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$'
    ).hasMatch(this);
  }

  bool get strongPassword {

    return length >= 8 &&
        contains(RegExp(r'[A-Z]')) &&
        contains(RegExp(r'[0-9]'));
  }
}

void main() {

  var email =
      'uliya@gmail.com';

  var password =
      'Uliya123';

  print(email.validEmail);

  print(password.strongPassword);
}

13.Ringkasan dan Best Practices

Extension Methods merupakan fitur pada Dart yang digunakan untuk menambahkan method baru pada tipe data yang sudah ada tanpa mengubah kode aslinya.

Penulisan extension menggunakan sintaks extension NamaExtension on TipeData dan dapat berisi method, getter, setter, maupun operator.

Extension juga mendukung generic types seperti List<T> sehingga dapat digunakan pada berbagai jenis data.

Dalam penggunaannya, sebaiknya extension diberi nama yang jelas dan disusun berdasarkan tipe data agar lebih rapi.

Programmer juga perlu menghindari penggunaan extension secara berlebihan serta memperhatikan penanganan null safety dengan baik.

Selain itu, penambahan komentar atau dokumentasi pada public extension dapat membantu kode lebih mudah dipahami.
